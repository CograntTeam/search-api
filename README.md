# Cogrant Search API

Partner-facing FastAPI gateway in front of Cogrant's n8n grant-search workflows.

## What this service does

Third-party partners hit a stable, authenticated HTTP API here. Each request
creates an async job, fires a webhook to n8n, and records the result in
Airtable when n8n calls back. Partners poll (or receive a webhook callback)
for the final result.

```
Partner client
   | HTTPS + API key
   v
[FastAPI: this repo]  --(webhook)--> [n8n Cloud workflows 1.0, 1.1A, 1.1B]
   ^                                          |
   +---- POST /internal/jobs/{id}/complete <--+ (shared secret)
```

## Tech stack

- FastAPI, Pydantic v2, uvicorn
- Airtable (via pyairtable) as the current job + key store — will migrate to
  Postgres once we exceed ~50 req/min sustained. The `AirtableRepo` interface
  is the seam.
- Deployed to Render from `main` branch; custom domain `api.cogrant.eu`.

## Local development

```bash
python -m venv .venv && source .venv/bin/activate
pip install -r requirements-dev.txt
cp .env.example .env
# fill in AIRTABLE_PAT, N8N_WEBHOOK_BASE_URL, INTERNAL_SHARED_SECRET
uvicorn app.main:app --reload
```

Visit http://localhost:8000/docs.

Run tests:

```bash
pytest
```

## Deployment

This repo has a `render.yaml` blueprint. On Render:

1. New → Blueprint → select this repo.
2. Set the secrets marked `sync: false` in the service's Environment tab
   (`AIRTABLE_PAT`, `N8N_WEBHOOK_BASE_URL`). `INTERNAL_SHARED_SECRET` is
   auto-generated by Render.
3. Settings → Custom Domains → add `api.cogrant.eu`.
4. At your DNS provider, add a CNAME `api` → `<service>.onrender.com`.
